Luzen Documentation

Luzen is a premium architecture & interior-design HTML template — GSAP-driven editorial motion, a hand-built WebGL hover layer, and a fully responsive 7-page layout. This guide covers everything you need to install, customize, and ship it.

01 — Overview

Introduction

Thank you for purchasing Luzen. The template targets architecture studios, interior designers, and creative practices that want a quiet, luxury editorial feel rather than a generic corporate look.

What's included

  • 7 HTML pages — Home, About, Services, Projects, Project Detail, Contact, Team Detail.
  • SCSS source organized in a 7-1 architecture, compiling to the served CSS.
  • Local vendor bundle — no runtime CDN dependency for scripts.
  • noImages preview package for fast evaluation without licensed photography.
  • This documentation.

Key features

  • GSAP 3 + ScrollTrigger editorial reveals & counters
  • Lenis smooth scrolling
  • Custom WebGL displacement hover (Three.js) on project imagery
  • Cross-fading hero slider with clip-path transitions
  • Magnetic buttons, custom cursor, film-grain texture layer
  • Animated skill bars & metric counters
  • Fully responsive, accessible markup
02 — Getting started

Installation

Luzen is a static HTML template — no build step is required to use it. Simply open the files or serve the folder.

Option A — Open directly

Unzip the package and open index.html in any modern browser. Because the hero uses a WebGL canvas and fetch-free assets, most browsers run it from file:// directly.

Option B — Local server (recommended)

A local server avoids any browser file-protocol restrictions. From the template root:

bash
# any one of these
npx serve .
# or
python3 -m http.server 8080
# then open http://localhost:8080

Option C — Recompile styles

Only needed if you edit the SCSS source. Install Dart Sass once, then build:

bash
npm install        # installs sass (devDependency)
npm run build:css  # compile scss → assets/css
npm run watch:css  # recompile on save
i

Google Fonts load over HTTPS. Everything else — scripts, styles, images — is local, so the template works fully offline once fonts are cached.

03 — Organization

File Structure

All assets live under assets/. Each top-level HTML page sits in the root for clean URLs.

luzen/ ├─ index.html # Home ├─ about.html ├─ services.html ├─ projects.html ├─ project-detail.html ├─ contact.html ├─ team-details.html ├─ package.json # SCSS build scripts ├─ assets/ │ ├─ css/ # compiled, served stylesheets │ │ ├─ vendor.css # Bootstrap grid │ │ ├─ main.css # global (every page) │ │ ├─ home.css · about.css · services.css … │ ├─ js/ │ │ ├─ vendor.js # GSAP+ST+CustomEase, Lenis, Three │ │ ├─ main.js · effects.js │ │ ├─ home.js · hero-slider.js · about.js … │ ├─ img/ │ │ ├─ hero/ · projects/ · team/ · about/ │ │ ├─ services/ · logos/ · icons/ · misc/ │ └─ scss/ # editable source (7-1 pattern) ├─ documentation/ └─ luzen-noImages/ # preview package w/ placeholders
04 — Markup

HTML Structure

Every page follows the same skeleton. Swap the page-specific CSS/JS in the head and before </body>; everything else is shared.

html
<head>
  <!-- fonts + grid + global, then this page's css -->
  <link rel="stylesheet" href="assets/css/vendor.css" />
  <link rel="stylesheet" href="assets/css/main.css" />
  <link rel="stylesheet" href="assets/css/about.css" />
</head>
<body>
  <div class="cursor-dot"></div>     <!-- custom cursor -->
  <header id="header"> … </header>
  <!-- page sections -->
  <footer id="footer"> … </footer>

  <!-- vendor bundle, then shared, then page script -->
  <script src="assets/js/vendor.js"></script>
  <script src="assets/js/main.js"></script>
  <script src="assets/js/effects.js"></script>
  <script src="assets/js/about.js"></script>
</body>
i

The brand lockup is a single flex unit: an <img class="logo-icon"> followed by <span class="logo-wm">Luz<span class="logo-e">e</span>n</span>. Keep the wordmark wrapped so letter-spacing stays even.

CSS Structure

Served CSS lives in assets/css/. Each page loads exactly three stylesheets: the vendor grid, the global main.css, and one page file.

Page CSS is intentionally kept separate. Each page tunes the shared [data-rv] scroll-reveal timing slightly differently — merging them into one file would flatten those differences.

06 — Source

SCSS Structure

The editable source lives in assets/scss/ and follows the 7-1 pattern. main.scss compiles to main.css; each pages/_*.scss partial compiles to its own page stylesheet.

scss/ ├─ abstracts/ # _variables _mixins _functions (no CSS output) ├─ base/ # _reset _typography _global ├─ layout/ # _header _footer _grid ├─ components/ # _buttons _cards _forms _accordion ├─ pages/ # _home _about _services _projects … ├─ utilities/ # _helpers (utilities + responsive) └─ main.scss # import manifest → main.css

Editing & compiling

Change a token in abstracts/_variables.scss, then recompile:

bash
npm run watch:css   # live recompile while you edit

Helper mixins and functions are available for your own additions:

scss
.feature {
  @include flex-center(column);   // from _mixins
  padding: rem(32);                // from _functions → 2rem
  @include bp($bp-sm) { padding: rem(20); }
}
07 — Behavior

JavaScript

Scripts are plain ES5+/vanilla — no framework, no bundler. Load order is always vendor → main → effects → page.

FileResponsibility
vendor.jsGSAP + ScrollTrigger + CustomEase, Lenis, Three.js (bundled)
main.jsLenis init, custom cursor, preloader, nav, shared reveals/counters
effects.jsWebGL displacement hover, magnetic buttons, film grain, cursor labels
home.jsHome editorial choreography (reveals, marquee, testimonials)
hero-slider.jsHero slider transitions, progress, drag/touch
about / services / projects / project-detail / contact / team-details.jsPer-page choreography
i

Each page script sets a flag (e.g. window.__LUZEN_ABOUT__) so main.js cedes reveal/counter control to the page module and they never double-fire.

08 — Motion

GSAP Animations

Motion is driven by GSAP 3 with ScrollTrigger. Entrance reveals use a fail-safe IntersectionObserver system (with a timed safety net) so content can never get stuck hidden, while counters and skill bars animate once on first view.

Reveal markup

Add data-rv to any element to opt it into the reveal system; add a direction class to taste:

html
<h2 data-rv>Fades up by default</h2>
<div data-rv class="rv-left">Slides in from left</div>
<figure data-rv class="rv-clip">Clip-path reveal</figure>

Tuning timing

Reveal transition timing lives at the top of each page CSS file under the [data-rv] selector. Adjust the transition duration / translateY distance there.

!

Keep the safety-net timeout in the page JS intact. It force-reveals any element that hasn't intersected, guaranteeing content is never invisible on tall pages.

09 — Make it yours

Customization

Colors

All colors are CSS custom properties in :root (top of main.css, or abstracts/_variables.scss). Change them once and the whole site updates.

Dark--dark #1C1C1D
Surface--surface #171719
Light--light #edece8
Accent--accent #CAA05C
css
:root {
  --accent: #CAA05C;   /* brand gold → change me */
  --dark:   #1C1C1D;
  --light:  #edece8;
}

Typography

Two families: Cal Sans (display) and Golos Text (body), loaded from Google Fonts in each page <head>. Swap the <link> and update --f-display / --f-body.

Spacing & type scale

Fluid sizes use clamp() via the --fs-h1 … --fs-h6 tokens. Edit them in one place to rescale every heading.

Components

Buttons (.btn-luzen), the gold CTA band (.page-cta), cards, and the accordion are all class-driven — copy the markup from any page and it inherits styling automatically.

Logo & Favicon

Logo usage

The brand lockup is one flex unit: the icon (assets/img/logos/logo-mini-light.svg) followed by the wordmark, with the E highlighted in brand gold. Keep the wordmark wrapped in .logo-wm so letter-spacing stays even across the icon-to-text gap.

html
<a href="index.html" class="header-logo">
  <img src="assets/img/logos/logo-mini-light.svg" class="logo-icon" alt="" />
  <span class="logo-wm">Luz<span class="logo-e">e</span>n</span>

Two icon variants ship in assets/img/logos/: logo-mini-light.svg (for dark backgrounds — header, footer, preloader) and logo-mini-dark.svg (for light backgrounds). To replace the logo, swap these SVGs or edit the wordmark text directly.

12 — Forms

Contact Form

The contact form is wired to Web3Forms — a free, no-backend form API. It works the moment you deploy: submissions are delivered by email with no server code to set up.

Overview

Web3Forms accepts a standard POST and emails you each submission. The template submits over AJAX (fetch) so the page never reloads and the custom success / error UI stays intact.

Current integration

The form posts to https://api.web3forms.com/submit. The submit handler lives in assets/js/contact.js (the initWeb3Forms() function) — it validates the fields, sends the data, then shows an inline status message and resets the form on success.

i

The template ships with a working demo Access Key so you can test immediately. Replace it with your own before launching a production site (see below).

Access Key location

The key is a single hidden input near the top of the <form> in contact.html:

html
<!-- contact.html · inside <form class="ct-form"> -->
<input type="hidden" name="access_key"
       value="1689852d-c68d-46fa-977e-51c4bc04d29a" />

Use your own key

  1. Create a free account at web3forms.com and enter the email where you want submissions delivered.
  2. Copy the Access Key they email you.
  3. Paste it into the value of the access_key input in contact.html — in both the main template and luzen-noImages/ if you use that copy.

Optionally edit the adjacent subject and from_name hidden inputs to customise the notification email.

Testing the form

Serve the site over http(s) (not file://, which blocks fetch), fill in the fields, and submit. A success message confirms delivery and the email arrives at your configured address. The first submission for a brand-new key must be confirmed via the verification email Web3Forms sends.

Spam protection

A hidden honeypot field (name="botcheck") is already included — bots that fill it are rejected automatically. Web3Forms also offers optional reCAPTCHA / hCaptcha in your dashboard for extra protection.

Troubleshooting

  • No email received — confirm the new key's verification email, and check spam. The demo key does not deliver to you.
  • Submission blocked locally — open the site via a local server, not file://.
  • Error message shown — open the console; a non-success response usually means an invalid or unverified Access Key.
  • Fields not validating — every required field must be filled; native validation runs before the request is sent.

14 — Going live

Deployment

Luzen is 100% static — there is no server runtime to configure. Upload the template root (the folder containing index.html and assets/) and you are live.

Shared hosting & cPanel

In cPanel open File Manager → public_html, upload the template zip, and extract it there. Ensure index.html sits at the web root (not inside a subfolder) for a clean domain URL.

FTP

With an FTP client (e.g. FileZilla), connect with your host credentials and upload the entire folder to public_html/ (or your domain root), preserving the directory structure exactly.

Vercel

bash
npm i -g vercel
vercel        # framework preset: Other · output dir: ./

Or drag-and-drop the folder into the Vercel dashboard.

Netlify

Drag the template folder onto app.netlify.com/drop, or connect a Git repo with publish directory = root and no build command.

GitHub Pages

Push the files to a repository, then Settings → Pages → Branch: main / root. Your site publishes at username.github.io/repo.

Any static host

Cloudflare Pages, AWS S3 + CloudFront, Firebase Hosting, Surge — all work the same way: upload the folder as-is.

i

Recommended: always serve over HTTPS, enable gzip/Brotli compression, and set long cache lifetimes on assets/ (see Optimization). Keep the folder structure intact so relative paths resolve.

15 — Performance

Optimization Tips

Images

  • Export at the dimensions the slot actually uses (see Image Replacement) — never ship a 4000px file into a 740px frame.
  • Prefer WebP / AVIF over JPG/PNG; the template already uses WebP where possible.
  • Run images through an optimizer (Squoosh, ImageOptim, TinyPNG) before upload.

Lazy loading

Gallery and below-the-fold images use loading="lazy". Add it to any new <img> that isn't in the first viewport.

Caching

apache
# .htaccess — cache static assets for a year
<FilesMatch "\.(css|js|woff2|webp|jpg|png|svg|ico)$">
  Header set Cache-Control "max-age=31536000, immutable"
</FilesMatch>

Minification

For production, compile SCSS minified and minify scripts:

bash
sass assets/scss/main.scss assets/css/main.css --style=compressed

Hosting & delivery

  • Use a host with HTTP/2 or HTTP/3 and a CDN.
  • Enable gzip/Brotli — it shrinks the vendor bundle dramatically.
  • Preconnect to Google Fonts is already in the <head>.
!

The largest asset is Three.js inside vendor.js (~590 KB uncompressed, far less gzipped). If you remove the WebGL hover effect, you can drop Three.js from the bundle for a much lighter payload.

16 — Fixes

Troubleshooting

CSS not loading

Confirm the assets/css/ paths resolve and that file names match exactly — most live hosts are case-sensitive (Main.css ≠ main.css). Make sure the assets/ folder was uploaded.

JavaScript not loading / animations dead

Scripts must load in order vendor → main → effects → page. Open the browser console: if gsap is not defined appears, vendor.js failed to load or is out of order.

Broken images

Check the assets/img/ path and case in the src, and confirm the image was uploaded. Moving an HTML file changes its relative path to assets/.

Missing fonts

Fonts come from Google Fonts over HTTPS. If a network blocks them, self-host the families and update the <link> in each page <head>.

Video won't play

The data-video value must be a YouTube embed URL (youtube.com/embed/ID), not a watch?v= URL — the latter is blocked inside iframes. Autoplay begins only after the play button opens the modal.

404 on asset paths

Paths are relative to each HTML file's location. Keep pages at the project root and the assets/ folder beside them; don't relocate one without the other.

Animation glitches

Ensure ScrollTrigger (in vendor.js) loads before page scripts. Users with reduce motion enabled get simplified transitions by design.

Changes not showing

Browsers cache CSS/JS aggressively. Hard-refresh (Ctrl/Cmd + Shift + R) or cache-bust with a query string (main.css?v=2) after updates.
